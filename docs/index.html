mksql (demo)(github)(feedback)

create SQL databases ... and query them with HTTP

what?

mksql is a new project to quickly create, explore, and publish SQL databases.

can I try it?

See a read-only demo here.

how does it work?

1 - you upload data - .csv, .tsv, .xls, JSON, Google Sheets, Airtble, etc.

2 - mksql makes a SQL database.

3 - You query the data with a Web UI or HTTP API.

use cases?

web UI - ad-hoc data exploration.

HTTP API - static websites, Jupyter/Observable/R notebooks, shell scripts, etc.

SQL injection?

The Web UI and HTTP API run in read-only mode

Who owns my data?

You do - mksql is self hosted and has an open source core (MIT License).

How does it make money?

There is (or, will be) a commerical license for pro / enterprise features.

The source code will be available under commerical license.

Why not X?

mksql meets some of my personal, idiosyncratic criteria.

  • own my data - or, limit the of parties I share with (e.g. only AWS)
  • open source - extend it, see how it works, resilient if company goes under, etc.
  • single node - core service should run on a single VPS / desktop / laptop with minimal configuration.
  • modest VPS performance - should handle ~100 requests / second & ~50 TB data sources on a single node
  • serverless - support publishing read-only sql to aws lambda / cloudflare workers, etc.
  • cli support - command line experience as important as web
  • curl-able - HTTP API should work well with curl & command line tools (jq, awk, etc)
  • minimal runtime dependencies - run from a single statically-linked executable
  • Golang - I like Go
